## What is a programing language:

**A programming language is** the bridge between human thinking and machine execution, enabling the creation of software, websites, games, and more.

Programming languages are like a multi-flavor yogurt pack:

- They all share the **same base ingredients**‚Äîvariables, loops, conditionals, functions...
    
- But each one has its own **flavor**: syntax, philosophy, strengths, and quirks.

For example:

- **Python** is like vanilla‚Äîsmooth, beginner-friendly, and goes with everything.
    
- **JavaScript** is like strawberry‚Äîpopular, versatile, but sometimes messy.
    
- **C++** is like dark chocolate‚Äîpowerful, intense, and not for the faint of heart.
    
- **Rust** might be mango‚Äînew, refreshing, and surprisingly robust.
    

So yes, they‚Äôre all part of the same ‚Äúpack,‚Äù but depending on your taste (or your project), you‚Äôll reach for a different one.

## Core Programming Concepts

Before diving into specific languages, here‚Äôs a quick overview of the basic programming concepts that will appear in each language section. These are the building blocks of most programming languages, even if the syntax changes.

### Printing Text

- How to display output on the screen.
    
- Examples: `print()` in Python, `echo` in Bash, `console.log()` in JavaScript.

### Input

- How to **receive data from the user** during program execution.
	
- Often combined with `print()` to create interactive programs.
	
- Input is typically stored in **variables** and used in conditionals, loops, or functions.

### Variables and Data Types

- **Variables** store values in memory.
    
- **Data types** define the kind of value: number, string, boolean, etc.
    
- Some languages require type declaration (static), others don‚Äôt (dynamic).

### Conditionals

- Making decisions: `if`, `else`, `switch`.
      
- Used to control program flow.
    
- Some languages support **ternary expressions** for inline decisions: `condition ? value_if_true : value_if_false`

### Loops

- Repeating actions: `for`, `while`, `foreach`.
	
- **Syntax and behavior** vary depending on the loop type

### Data Containers

- **Lists / Arrays**: ordered values ‚Üí `[1, 2, 3]`
    
- **Dictionaries / Maps**: key-value pairs ‚Üí `{"name": "Anna"}`
    
- **Tuples**: ordered, immutable ‚Üí `(1, 2)`
    
- **Sets**: unique, unordered values ‚Üí `{1, 2, 3}`

> Used to organize and access data efficiently.

### Try...Catch

- Used to control the program and revoke possible errors

### Functions

- Reusable blocks of code.
    
- Defined with `def`, `function`, or other keywords depending on the language.

### Comments

- Notes inside code that are ignored by the computer.
    
- Syntax: `#`, `//`, `/* */`, depending on the language.

### Imports & Packages

- Imports are used to import usefull packages like `os` in python

### Object-Oriented Programming

Object-Oriented Programming is like organizing your code into little ‚Äúsmart boxes‚Äù that know how to behave.

Imagine building with LEGO:

- Each **object** is a LEGO piece with its own shape (data) and behavior (methods).
    
- You can **reuse** pieces, **combine** them, or **customize** them to build something bigger.
    
- OOP helps make code more **modular**, **reusable**, and **easier to maintain**‚Äîespecially in large projects.   

> ‚öÝÔ∏è Not all programming languages support OOP. Some are procedural (like C), functional (like Haskell), or logic-based (like Prolog). Others, like Python or JavaScript, support multiple paradigms including OOP.

Key concepts:

| Concept           | Description                                                                               |
| ----------------- | ----------------------------------------------------------------------------------------- |
| **Class**         | The blueprint for creating objects. Like a mold for LEGO bricks.                          |
| **Object**        | An instance of a class. A specific LEGO brick made from the mold.                         |
| **Encapsulation** | Hiding internal details. Only exposing what‚Äôs necessary. Like a sealed box.               |
| **Inheritance**   | Passing down traits from one class to another. Like family genetics.                      |
| **Polymorphism**  | One interface, many forms. Like a remote that controls different devices.                 |
| Abstraction       | Showing only essential features. Like driving a car without knowing the engine mechanics. |
